# Bilibili Guardian

一个给 Bilibili 用户使用的清理工具，把“推荐流过滤”和“视频广告识别/自动跳过”整合进同一套页面内控制中心。

- 当前版本：`0.2.0`
- 安装形态：Chrome / Edge Manifest V3 扩展、Tampermonkey 单文件 userscript
- 适用页面：`https://www.bilibili.com/*`
- License：`MIT`
- 作者：`StarsWhere`

## 核心能力概览

### 推荐流过滤

- 清理首页、搜索、热门、排行榜和分区类页面中的广告卡片与直播卡片
- 按分类、标题关键词、UP 主关键词做进一步过滤
- 支持持续监听页面变化并自动重扫
- 所有过滤行为都可以在控制中心里开关和调整

### 视频广告识别与自动跳过

- 在视频页自动识别当前 `BV` 号
- 获取弹幕、评论区首条或置顶评论作为参考信息
- 调用你配置的 AI 服务判断视频里是否存在广告片段
- 命中阈值后自动把播放器跳到广告片段之后
- 支持缓存命中、手动重跑和当前视频单独开关自动跳过

## 适用人群与不适用场景

### 更适合哪些用户

- 经常刷 Bilibili 首页、搜索页、热门页，希望减少噪音内容的用户
- 已有 AI API Key，想让视频页自动识别“恰饭”片段的用户
- 希望统一在一个面板里管理过滤、识别、排错和 AI 配置的用户
- 更偏好扩展安装，或更偏好 Tampermonkey 快速导入的用户

### 哪些场景不太适合

- 不打算配置任何 AI API Key，但又希望使用视频广告识别的人
- 需要 100% 准确识别广告区间的人
- 主要使用 `www.bilibili.com` 之外页面的人
- 不希望扩展或脚本访问 Bilibili 页面内容、弹幕、评论信息和你配置的 AI 接口的人

## 安装方式

### 方式一：从 GitHub Release 下载

如果你只想尽快安装，不想自己构建，优先使用 Release。

1. 打开 [GitHub Releases](https://github.com/StarsWhere/Bilibili-Guardian-Extension/releases)
2. 下载最新版本中的扩展包或 userscript 单文件
3. 根据你选择的安装形态继续操作

可下载的正式产物包括：

- 扩展版压缩包：`bilibili-guardian-extension-vX.Y.Z.zip`
- 扩展版校验文件：`bilibili-guardian-extension-vX.Y.Z.zip.sha256`
- 油猴脚本：`bilibili-guardian.user.js`

### 方式二：从源码构建浏览器扩展

适用场景：

- 你主要使用 Chrome 或 Edge
- 你希望长期启用扩展版
- 你想跟随仓库最新代码自行构建

安装步骤：

1. 安装依赖

   ```bash
   npm install
   ```

2. 构建扩展产物

   ```bash
   npm run build
   ```

3. 打开扩展管理页：Chrome 使用 `chrome://extensions`，Edge 使用 `edge://extensions`

4. 打开右上角“开发者模式”
5. 选择“加载已解压的扩展程序”
6. 选择项目里的 `dist/` 目录

扩展版实际产物为：

- `dist/manifest.json`
- `dist/content.js`
- `dist/background.js`

### 方式三：构建并安装 Tampermonkey userscript

适用场景：

- 你已经在浏览器中使用 Tampermonkey
- 你希望用单文件脚本快速导入
- 你更习惯 userscript 的安装和管理方式

安装步骤：

1. 安装 Tampermonkey
2. 安装依赖并构建脚本

   ```bash
   npm install
   npm run build:userscript
   ```

3. 打开 `dist/bilibili-guardian.user.js`
4. 在 Tampermonkey 安装页确认导入
5. 首次请求自定义接口时，如果脚本管理器提示授权，请允许目标域名

userscript 实际产物为：

- `dist/bilibili-guardian.user.js`

## 首次配置与快速上手

第一次使用时，建议按下面顺序完成初始化。

1. 打开任意 `https://www.bilibili.com/*` 页面
2. 点击页面内悬浮按钮，打开控制中心
3. 先完成 AI 服务设置：`Provider`、`Base URL`、`API Key`、`模型名称`
4. 再根据需要调整推荐流过滤规则
5. 最后确认视频识别阈值和默认自动跳过开关

如果你只是想先体验：

- 推荐流过滤可以直接使用默认配置
- 视频识别至少需要先填好 `API Key` 和模型相关配置
- 默认会优先启用自动跳过，因此第一次配置后建议随便打开一个视频看一下识别结果是否符合你的预期

## 日常使用说明

### 首页、搜索、热门和排行榜过滤如何生效

只要你进入支持的推荐流页面，扩展就会自动扫描当前卡片列表。命中过滤规则的卡片会被移除；如果页面后续继续加载新内容，开启“持续扫描”后会自动再次清理。

适用范围包括：

- 首页
- 搜索结果页
- 热门页
- 排行榜页
- 分区 / 频道类页面

常用规则包括：

- 过滤广告
- 过滤直播
- 分类黑名单
- 标题关键词黑名单
- UP 主关键词黑名单

### 视频页会发生什么

进入视频页后，扩展会先识别当前 `BV` 号，再获取视频 `CID`、弹幕和评论区参考信息。随后它会调用你配置的 AI 服务，让模型返回结构化结果，再根据阈值、片头保护窗口和最长可跳片段决定是否建议跳过或直接自动跳过。

你会经历的常见状态包括：

- 正在整理参考信息
- 正在调用 AI 识别
- 已直接使用上次识别结果
- 识别完成
- 已自动跳过
- 本次识别失败

### 缓存命中时是什么体验

同一个视频在缓存有效期内再次打开时，扩展会优先使用上一次识别结果，而不是重新发起 AI 请求。这样可以减少重复请求，也能让视频页更快进入可跳过状态。

默认缓存有效期为 `240` 分钟。命中缓存时，你会看到“已直接使用上次识别结果”之类的提示。

### 如何手动重新识别

如果你觉得当前结果不理想，或者刚刚调整了阈值和提示词，可以手动重新跑一次分析。

可用入口包括：

- 视频快捷卡里的“重新识别”
- 控制中心视频相关区域里的手动触发按钮
- 诊断和排错后再次返回视频页重新执行

## AI 服务配置说明

### 支持的 Provider

- OpenAI
- DeepSeek
- Gemini
- Anthropic
- 自定义 OpenAI-Compatible 接口

### 各 Provider 的配置思路

| Provider | 默认 Base URL | 模型获取方式 | 说明 |
| --- | --- | --- | --- |
| OpenAI | `https://api.openai.com/v1` | 可尝试自动拉取 | 适合直接填写官方接口或兼容 OpenAI 的托管服务 |
| DeepSeek | `https://api.deepseek.com/v1` | 可尝试自动拉取 | 与 OpenAI 兼容风格接近 |
| Gemini | `https://generativelanguage.googleapis.com/v1beta` | 可尝试自动拉取 | 通过 Gemini 模型列表接口获取候选模型 |
| Anthropic | `https://api.anthropic.com/v1` | 使用内置推荐列表 | 当前更偏向手动选择模型名称 |
| Custom | 由你填写 | 可尝试自动拉取 | 需要完整 `http://` 或 `https://` 地址 |

### 自定义 OpenAI-Compatible 接口

使用自定义接口时，请确认以下几点：

- `Base URL` 必须是完整地址，例如 `https://example.com/v1`
- 必须填写有效的 `API Key`
- 必须填写有效的模型名称
- 扩展版会在保存或请求时申请该域名的额外访问权限
- Tampermonkey 版会在首次外部请求时由脚本管理器要求授权

### 模型拉取与权限说明

控制中心支持尝试拉取模型列表，但不同 Provider 的能力不同。

- OpenAI、DeepSeek 和自定义兼容接口会尝试访问 `Base URL/models`
- Gemini 会尝试访问模型列表接口
- Anthropic 当前以内置推荐模型列表为主
- 如果拉取失败，界面会回退到内置默认模型建议

### 识别偏好与阈值

除了 API 参数，你还可以继续调整识别行为，例如：

- 自动跳过阈值
- 片头保护窗口
- 最长可跳片段
- 请求超时
- 弹幕白名单 / 黑名单
- 自定义提示词

## 权限、隐私、局限与免责声明

### 扩展和脚本会访问什么

为了完成过滤和视频识别，项目会访问：

- `https://www.bilibili.com/*` 页面本身
- Bilibili 的视频信息和弹幕接口
- 页面内可见的评论区首条或置顶评论内容
- 你主动配置的 AI Provider 接口

### API Key 和本地存储

- `API Key` 由你自行填写
- 扩展版配置保存在浏览器的 `chrome.storage.local`
- userscript 版配置保存在脚本管理器提供的本地存储中
- 项目不会替你托管或代管 API Key

### 自定义接口权限

- 扩展版对自定义接口会按域名申请额外权限
- 如果你拒绝了域名权限申请，请求会失败
- Tampermonkey 版对自定义接口依赖脚本管理器的外部请求授权

### 已知局限

- AI 识别存在误判和漏判，不保证每次都能识别出广告
- 返回的开始时间和结束时间可能不够精确
- 当有效弹幕过少时，扩展可能直接跳过 AI 分析
- Bilibili 页面结构变化后，部分卡片选择器或交互可能需要更新
- 项目不保证对所有页面结构长期稳定适配

### 使用前建议

- 先用一个你熟悉的视频测试识别效果
- 如果你的接口计费敏感，优先观察缓存命中效果和手动重跑频率
- 对自动跳过比较谨慎时，可以先关闭默认自动跳过，只查看识别结果

## 常见问题

### 页面上没有看到悬浮按钮

请先检查：

- 当前页面是否在 `https://www.bilibili.com/*`
- 扩展是否已经成功加载，或 userscript 是否已经启用
- 页面是否被其他脚本、样式或浏览器插件干扰

### 视频识别时报“请先配置 AI API Key”或类似错误

通常表示以下字段还不完整：

- `API Key`
- `模型名称`
- `Base URL`

如果你使用的是自定义兼容接口，还要继续检查：

- `Base URL` 是否是完整 URL
- 是否已经授予扩展域名权限或 Tampermonkey 外部请求授权

### 为什么有时不会触发自动跳过

常见原因包括：

- 当前识别结果的概率没有达到阈值
- AI 没有返回可用的开始时间和结束时间
- 有效弹幕过少，系统直接跳过了 AI 分析
- 该视频命中了缓存，但缓存结果本身不满足自动跳过条件

### 油猴版导入后无法请求外部接口

请优先确认：

- 是否使用 Tampermonkey
- 脚本是否已启用
- 首次访问外部接口时是否拒绝了授权提示
- 自定义接口服务端是否允许当前请求方式

### 拉取模型列表失败怎么办

模型拉取失败并不一定意味着接口完全不可用。你可以：

- 先手动填写模型名称
- 检查 `Base URL` 是否正确
- 检查 `API Key` 是否有效
- 检查是否已经授予了对应域名权限

### 想看更详细的排错信息怎么办

控制中心内置了诊断区域，会记录最近的日志、错误提示和部分 AI 返回预览。遇到结构化解析失败、响应异常或权限问题时，优先打开问题排查区域查看。
